ev text editor

ev is a simple cross-platform text editor written in C++ with minimal dependencies. The executable is just a few hundred kilobytes in size. The editor works in a text console or terminal on five platforms - Windows, macOS, Linux, Solaris and AIX. GUI support is planned for the next major version.

If you think that ev stands for "eradicate vi" then you are mistaken :-) I have great respsect for that venerable text editor. I personally used vim as my main development environment for several years and I love it passionately!

Supported features

  • basic text editing operations
  • multiple open documents
  • open/save documents
  • copy/delete/paste
  • find/replace
  • autoindent
  • indent/unindent
  • toggle comment
  • go to line
  • cycle between recently edited locations
  • ability to execute make and use ev as a simple IDE
  • basic Unicode support (can read/write UTF-8 and UTF-16, aware of multibyte characters)
  • autocomplete based on words in open documents
  • macro recording/playback
  • syntax highlighting

building with make: by default ev executes 'make' on UNIX systems and 'nmake.exe' on Windows. This can be overriden by setting MAKE_CMD environment variable.

macOS note: By default Mac Terminal app sends weird codes for alt+key combinations. They are currently unsupported by ev. You can fix the issue by turning on 'Use Option as Meta key' on the Keyboard tab in the terminal profile settings. To make ev more useful you can add a few key mappings there:

End - \033[4~
Option+End - \033\033[4~
PgUp - \033[5~
PgDn - \033[6~
Option+Delete - \033\033[3~

Commands

Command line is accessible at the bottom of the screen by pressing F2 key. Optional parameters are in square brackets.

s - save current document

sa - save all documents

c - close current document

q - quit editor

qs - save all documents and quit

g number - go to line number

f[i] search-string - find string
i - ignore case

r[ida] search-string replace-string - replace string
i - ignore case
d - replace all matches in current document
a - replace all matches in all documents
use any character as string separator instead of space

n filename - new file

o filename - open file

Autocomplete

Autocomplete works by scanning for all identifiers in open documents and it lets you complete words as you type by pressing Tab key. The list of autocomplete suggestions can be refreshed by saving documents or by pressing alt+'. Completetion works best with at least two starting letters. When you press Tab the current suggestion is inserted into the text but the cursor remains after the last letter typed. If there're letters to the right of the cursor, the inserted word overwrites them.

If you are satisfied with the suggestion, you can type the next character following the word such as space or punctuation. The cursor jumps to the end of the word and that character is inserted, minimizing the number of keystokes. If you move the cursor instead, the 'jump to end' feature is disabled and the next character will be inserted where the cursor is.

If you are not satisfied with the suggestion, you can press Tab or Shift+Tab to go to the previous/next suggestion or you can type one or more letters until autocomplete gives you the expected result without pressing Tab too many times. You can also press Backspace to erase previous letter and automatically suggest a new word based on the shorter prefix.

Syntax highlighting

Languages for which syntax highlighting is currently supported: C, C++, UNIX shell scripts. ev by default assumes bright screen background and uses darker colors for syntax highlighting to improve contrast. To override this you can use --dark command line option.
